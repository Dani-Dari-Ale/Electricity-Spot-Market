Easy Steps to Create an Executable in Python Using PyInstaller

Guide to create a Python executable file using PyInstaller.

Renu Khandelwal
Jul 30, 2020 · 7 min read

In this article, you will learn to

  • Install PyInstaller,
  • Understand what does PyInstaller do to create an executable
  • Create a simple executable
  • Add data files used by the executable
  • Add additional imports to the executable
  • Understand .Spec file and its usage

Installing PyInstaller

Installing PyInstaller is easy and is very similar to installing any python library.

pip install PyInstaller

You can check the version of PyInstaller.

pyinstaller --version

I am using PyInstaller version 3.6.

Creating an executable using PyInstaller

PyInstaller bundles Python application and all its dependent libraries into one package, and it does this by

  • Reading the script file.
  • Analyzing the python code to identify all the dependent libraries required for the execution
  • Creates a Spec file that contains the script name, dependent libraries, any data files, including different options that you provide to PyInstaller command.
  • Collect copies of all those libraries/files along with active python interpreter
  • Creates a BUILD folder in the same folder as the script and writes some log files and working files in the BUILD folder.
  • Creates a DIST folder in the same folder as the script, if it does not already exist.
  • Writes required dependent files along with the python script, either in one folder or in a single executable.

If you use command option onedir or -D while generating the executable, then executable is bundled in one-folder. This is the default mode. If you use onefile or -F option while creating the executable, then we have one executable, and all dependencies are bundled in a single executable file.

I have a simple python script in simple.py containing the following code.

name = input(" Enter your Name  ")
print(" Your name is ", name)

We will create an executable bundled into one file.

pyinstaller  --onefile  simple.py

Once the installation is successful, you have two new folders created called BUILD and DIST, along with a new file with a .spec extension. The .Spec file has the same name as the python script file.

PyInstaller creates a distribution directory, DIST containing the main executable and the dynamic libraries bundled in an executable file.

Executing the simple.exe file

What if you need to add some data files that will be used by the executable?

Adding data files to the executable

I have a CSV file called data.csv, and the Python script file counts the number of records in the data.csv file. We want to bundle the data.csv along with the bundled executable. The python script is named as simple1.py.

import numpy as np
import pandas as pddef count_records():
    data=pd.read_csv('data.csv')
    data_cnt=len(data)
    print("Total records : ", data_cnt)
   
    
if __name__ == "__main__" : 
    count_records()

We will create an executable bundled into one directory.

pyinstaller --add-data "data.csv;." simple1.py

The add-data option allows you to add data files that need to be bundled with the executable. This option can be applied multiple times

The syntax for add-data

— add-data <Source;Destination> on windows

— add-data <Source:Destination> on Linux

We see the data.csv file is now added to the DIST folder along with the executable.

When you open simple.Spec file you can see that the datas section has the data.csv file that will be copied to the current directory.

Running the simple1.exe file.

How to use onefile option and also add the data files to the executable?

When we specify the onefile option, to the PyInstaller, it unpacks all the files to a TEMP directory, executes the file, and later discard the TEMP folder. When you specify the add-data option along with onefile, then you need to read the data referred to in the TEMP folder. The path of the TEMP folder keeps changing and creates a _MEIxxxxxx-folder.

I have updated the Python script to read the TEMP folder and then access the datafile.

import numpy as np
import pandas as pd
import sys
import osos.chdir(sys._MEIPASS)
data_path= 'data\\data.csv'def count_records():
    data=pd.read_csv(data_path)
    data_cnt=len(data)
    print("Total records : ", data_cnt)
    
    
if __name__ == "__main__" : 
    count_records()

I created the executable using onefile and add-data option.

pyinstaller --onefile --add-data "data.csv;data" simple1.py

After the executable was created successfully, we see simple1.exe in the DIST folder.

I have copied the Simple1.exe to my desktop and executed it to check if it will pick the data.csv without giving “File not found error.”

Add additional imports to the executable using Hidden Imports.

The executable needs all the imports required for the Python script to work. Pyinstaller, at times, fails to identify dynamic imports or second level imports in the Python script, and in that case, when you run the executable, you will get an error. “ImportError: No module named..."

To solve the Import Error for the mentioned library, include the missing import library to the hidden-import list.

For example, to include the “os” library, we can use hidden imports, as shown below.

pyinstaller -F — add-data “data.csv;data” — hidden-import ‘os’ simple1.py

Understanding the spec file and its usage

Spec file is the first file PyInstaller builds to encode the Python script file and all the options you provide at the command prompt.

PyInstaller reads the contents of the Spec file to build the executable defining all the files needed for the executable to run along with additional settings.

The .Spec file is stored in the current directory by default with an extension “.spec.”

If you have any of the below requirements, then you can modify the spec file.

  • Bundle data files along with the executable
  • To Include runtime libraries like .dll file or .so files.
  • To bundle multiple programs using common libraries.

I have a Python script “simpleModel.py” that uses Tensorflow and prints the TensorFlow version.

import numpy as np
import pandas as pd
import tensorflow as tfdef View_model():
    print(tf.__version__)if __name__ == "__main__" : 
    model= View_model()

Compile the model using PyInstaller

pyinstaller -F simpleModel.py

After the model is compiled successfully, run the executable, and you can see an ImportError, as shown below.

So let’s fix the issue by updating the spec file.

One of the solutions to fix the Tensorflow issue is to create the Spec file.

pyi-makespec simpleModel.py -F

pyi-makespec command creates a default spec file containing all the options that we can specify at the command prompt. The simpleModel.spec file is created in the current directory.

As we have used the onefile option using -F, we will only see the exe section inside the Spec file.

If we use the onedir or default option, then you can see the collect section along with the exe section inside the Spec file.

You can open the simpleModel. Spec file and add the following code highlighted and Italicized to create hooks.

# -*- mode: python ; coding: utf-8 -*-block_cipher = None
import os
spec_root = os.path.realpath(SPECPATH)options = []
# options = [('v', None, 'OPTION')]from PyInstaller.utils.hooks import collect_submodules, collect_data_files
tf_hidden_imports = collect_submodules('tensorflow_core')
tf_datas = collect_data_files('tensorflow_core', subdir=None, include_py_files=True)a = Analysis(['simpleModel.py'],
             pathex=['\\test_model'],
             binaries=[],
             datas=tf_datas + [],
             hiddenimports=tf_hidden_imports + [],
             hookspath=[],
             runtime_hooks=[],
             excludes=[],
             win_no_prefer_redirects=False,
             win_private_assemblies=False,
             cipher=block_cipher,
             noarchive=False)

We are creating hooks and adding those hooks to the hidden imports and data section.

Hooks

Hook files extend the PyInstaller’s ability to handle special needs to include additional data required by the executable or need the ability to import dynamic libraries.

Typically Python packages use normal methods to import their dependencies, but there are few packages like Tensorflow that need to import dynamic libraries. PyInstaller cannot find all the libraries, and in some cases, it many have identified too many dependencies; in such a scenario, you can use the import helper method from PyInstaller.utils.hooks and collect al the submodules for the library.

Compile the model after updating the simpleModel. Spec file.

pyinstaller simpleModel.spec

Copied the executable to desktop, and it was now running successfully to display the Tensorflow version as 2.1.0.

Conclusion:

PyInstaller provides multiple options to create a simple to the complex executable for a Python script.

  • Executable can bundle all the required data using add-data option.
  • Executable and its dependent data and libraries can be bundled in one executable or a folder using onefile or onedir option, respectively.
  • Dynamic import and second level libraries can be included using hidden-imports.
  • Spec file allows you to create executable handling hidden import and custom data files and provides usage of hooks.
